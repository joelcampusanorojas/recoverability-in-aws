The object of the RTO is mesure the time that the application is down (for any cause) and 
this time is not causing significant damage to the business. The RPO is the data loss for 
the application is down and the time for data recovery in the application. 

For a single region outage, the RTO is the time it takes to start up a new database and this is arround 
5 minutes. The RPO the time take in promote the database for read and write is about 10 minutes.
This is a diferente in the single AZ outage the RTO to start up instance is ±2 minutes and the RPO is ±5 minutes.
